# Story Format Rules

正文文件的格式规范。当用户直接编辑正文文件时自动加载。

## 绝对禁止

1. **禁止段落超过 60 字**：每段不超过 60 个汉字（对话和内心独白除外）。手机阅读每行约15字，4行已是较大段落。一个注意力瞬间内的 2-3 句连续动作/感知/反应合为一段，段内用逗号串联，段间用换行切换镜头。禁止把同一镜头内的连续微动作按句号机械拆散。
2. **禁止段落间空行**：正文段落之间不加空行，保持紧密节奏。
3. **禁止炫技式对话标签**：基础标签「说」「问」「道」允许作为对话节奏标点。禁止「说道」「叹道」「厉声道」「柔声说」等刻意替换式标签——对话的情绪和语气应靠对话内容本身和上下文动作传递，不靠标签堆砌。
4. **禁止大段描写堆砌**：描写必须穿插动作或对话，不连续超过 3 段纯描写。
5. **禁止跳出视角**：已确定视角（第一人称/限知第三人称）后，不切换到其他角色内心。

## 格式规范

- 对话独立成行，用冒号或动作引出
- 段落以情绪或动作为单位划分
- 章节之间用 `---` 分隔
- 章节标题用 `## 第X章 章名` 格式

## Examples

### Correct — 注意力瞬间合段
```
沈栀抬手，灵力从指尖涌出，面前的结界出现一道裂缝。
碎纹像蛛网般蔓延开，她咬紧牙关加大输出，整条手臂开始发颤。
```
第1段：抬手→灵力涌出→裂缝，同一镜头内3个连续动作用逗号串联。第2段：碎纹蔓延→加大输出→手臂发颤，新镜头开始。

### Wrong — 注意力瞬间被机械拆碎
```
沈栀抬手。灵力从指尖涌出。
面前的结界出现一道裂缝。碎纹像蛛网般蔓延开。
她咬紧牙关加大输出。整条手臂开始发颤。
```
同一镜头内的微动作被机械按句号拆成碎片，读者注意力反复切碎。

### Wrong — 段落超过 80 字
```
沈栀抬手灵力从指尖涌出面前的结界出现一道裂缝碎纹像蛛网般蔓延开她咬紧牙关加大输出整条手臂开始发颤最后结界轰然碎裂碎片向四周飞溅她被冲击波震退撞在墙上胸口一闷嘴里泛出血腥味。
```
单段超过 80 字，信息密度过高，读者无法停顿。

### Correct — 对话基础标签
```
沈栀将茶杯往桌上一顿。
"你到底想说什么？"
陆衍没接话，只是看着窗外。
"没想说什么。"
```
对话通过动作和上下文引出，也可使用基础标签「说」「问」「道」。

### Wrong — 炫技式对话标签
```
沈栀将茶杯往桌上一顿。
她厉声说道："你到底想说什么？"
陆衍没接话，轻轻叹道："没想说什么。"
```
「厉声说道」「轻轻叹道」属于炫技式标签，对话的情绪应靠内容本身和上下文传递。

### Correct — 章节标题后无多余空行
```
## 第二章 暗流

陆衍推开门时，屋内已经坐了三个人。
沈栀坐在最里面的角落，手里捏着一张纸条。
```
章节标题 `##` 与正文之间仅一个空行（Markdown 渲染需要），正文段落间无空行。

### Wrong — 段落间多余空行
```
## 第二章 暗流


陆衍推开门时，屋内已经坐了三个人。


沈栀坐在最里面的角落，手里捏着一张纸条。
```
段落之间出现多余空行，破坏紧凑节奏。
